# Power BI Dashboard Design — Казпочта
**Данные:** март–май 2025 | 57 013 транзакций | 23 филиала | 7,07 млрд тг выручки

---

## Архитектура: 3 страницы с drill-through

```
┌─────────────────────────────────────────────────────────┐
│  Page 1: Executive Overview         [динамический заголовок]
│  "Выручка Казпочты — [выбранный период / филиал]"       
├─────────────────────────────────────────────────────────┤
│  Page 2: Филиальная аналитика       [Drill-through из P1]
│  "Детали — [FILIAL_NAME]"                               
├─────────────────────────────────────────────────────────┤
│  Page 3: Продуктовый Deep Dive      [Drill-through из P2]
│  "Продукты — [PRODUCT] в [FILIAL_NAME]"                
└─────────────────────────────────────────────────────────┘
```

---

## Page 1: Executive Overview

### Slicers (синхронизированы через все страницы)
- PERIOD (2025_3 / 2025_4 / 2025_5) — чипсы с мультиселект
- DIRECTION (РК / МЖД) — toggle-кнопки
- SNDRCTG (Физ. лицо / Юр. лицо) — radio buttons

### KPI Cards (верхняя строка)
| Карточка | Мера | Тренд |
|---|---|---|
| Выручка (тг) | `[Total Revenue]` | vs предыдущий период |
| Кол-во операций | `[Total CNT]` | MoM % |
| Средний чек | `[Avg Summa]` | vs план |
| Активных филиалов | `[Active Filials]` | — |

### Визуализации
1. **Waterfall Chart** — декомпозиция выручки по продуктам (Посылки → EMS → Письм.кор. → КГПО → Прочие)
2. **Matrix с Conditional Formatting** — Филиал × Продукт × Выручка (heatmap: красный=низко, зелёный=высоко)
3. **KPI Gauge** — выполнение плана выручки по всей компании (если есть план)
4. **Bar Chart** — топ-10 филиалов по выручке (горизонтальный), с drill-through на Page 2
5. **Line Chart** — динамика выручки по периодам: Посылки / EMS / Письм.кор.

## Page 2: Филиальная аналитика (Drill-through)

Открывается при правом клике на филиал в Page 1.
Динамический заголовок: `"Аналитика — " & SELECTEDVALUE(dim_filials[FILIAL_NAME])`

### Визуализации
1. **Scatter Plot** — Выручка vs Кол-во операций по продуктам (размер пузыря = средний чек)
2. **Stacked Bar** — структура выручки: РК / МЖД по месяцам
3. **Stacked Bar** — структура по категории клиента: Физ. / Юр. лицо
4. **Decomposition Tree** — разбивка выручки: Продукт → Подкатегория → ГК счёт
5. **Key Influencers** — что влияет на высокий средний чек в данном филиале
6. **Custom Tooltip** — при наведении на бар: топ-3 продукта по выручке

---

## Page 3: Продуктовый Deep Dive

Drill-through из Page 2 по продукту.

### Визуализации
1. **Pareto Chart** (линия + столбцы) — счета ГК по выручке с кумулятивной линией (80/20)
2. **Matrix** — Счёт ГК × Филиал × Выручка (тепловая карта)
3. **MoM Trend Lines** — помесячная динамика по выбранному продукту
4. **Q&A Visual** — "Задайте вопрос данным" (natural language)

## Performance оптимизация

| Аспект | Решение |
|---|---|
| Aggregation Tables | Создать агрегацию на уровне Филиал×Продукт×Период (69 строк vs 57k) — загружается в 1 сек |
| Import vs DirectQuery | **Import** — данные обновляются ежемесячно, latency не критична |
| Query Folding | Все трансформации в Power Query должны сворачиваться в SQL (проверить через View Native Query) |
| Row-Level Security | Руководитель ОФ видит только свой регион: DAX роль `[FILIAL_NAME] = USERPRINCIPALNAME()` |
| Mobile Layout | Page 1 адаптирован: 2 KPI карты + 1 bar chart на мобильном |
